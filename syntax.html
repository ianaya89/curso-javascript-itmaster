Scope

Es el ámbito y contexto de una variable.

El ámbito de una variable es el lugar dentro de un programa en el cual dicha variable vive y por lo tanto, donde puede ser usada.

Significa que una variable definida dentro de una función (con la palabra reservada var) va a poder ser usada dentro de esa función, pero no fuera de la misma.

Las variables declaradas fuera de una función (o sin la palabra var) son denomidas globales y seran accesibles en todos lados.

Scope

        
          // Ejemplo 1
          var value = 'global';

          function showValue() {
            alert(value); //global;
          }

          // Ejemplo 2
          var value = 'global';

          function showValue() {
            var value = 'private';
            alert(value); // private;
          }

          // Ejemplo 3
          function showValue() {
            var value = 'private';
            alert(value); // private;
          }

          alert(value); // ERROR

Closure

En JavaScript podemos definir funciones dentro de otras funciones, pudiendo acceder desde las funciones mas internas al scope de las mas externas (no al reves).

        
          function addAndIncrement(x, y) {
            var z = x + y;

            function increment() {
              z += 1;
            }

            increment();

            return z;
          }

          addAndIncrement(5, 5); // 11
          addAndIncrement(1, 1); // 3

Object.prototype

Todos los objetos de JavaScript estan linkeados a un objeto prototype.

Nos sirve para definir y luego extender comportamiento hacia otras clases (herencia).

Usando Constructores

        
          var Person = function(name, id) {
            this.name = name;
            this.id = id;
          }

          Person.prototype.sayHello = function() {
            return 'Hello my name is: ' + this.name;
          }

          var john = new Person('John', 14);
          var jane = new Person('Jane', 15);

          alert(john.sayHello());
          alert(jane.sayHello());

Herencia Pseudoclassical

        
          // Objeto Padre
          var Person = function() {
              this.canTalk = true;
          };

          // Definicion de metodos
          Person.prototype.greet = function() {
            if (this.canTalk) {
                console.log('Hi');
            }
          };

          // Objeto Hijo 1
          var Employee = function(name, title) {
              this.name = name;
              this.title = title;
          };

          // Aplicacion de Herencia
          Employee.prototype = new Person();

          // Definicion de metodos
          Employee.prototype.greet = function() {
            if (this.canTalk) {
                console.log('I am ' + this.name);
            }
          };

          // Objeto Hijo 2
          var Customer = function(name) {
              this.name = name;
          };

          // Aplicacion de Herencia
          Customer.prototype = new Person();

          // Instancias
          var bob = new Employee('Bob','Constructor');
          var joe = new Customer('Joe');
          var rg = new Employee('John','Developer');
          var mike = new Customer('Mike');

          bob.greet();
          joe.greet();
          rg.greet();
          mike.greet();

Array

Estructura de memoria lineal donde los datos son accesibles mediante indices.

Son menos eficientes que los arrays de otros lenguages de programacion porque en realidad son objetos.

Tanto la longitud del array como sus valores son mutables y el primer indice siempre es es 0.

Tips de Sintaxis

  • No se tienen en cuenta los espacios en blanco ni los saltos de linea (Es importante el indentado)
  • Es Case Sensitive
  • Usamos camelCase (salvo en los constructores)
  • No se define el tipo de dato de las variables
  • No es obligatorio el ';' al final de cada sentencia (Pero usenlo SIEMPRE!!!)
  • Comentarios en linea con // Comentario
  • Comentarios multilinea con /* Comentario */
  • Se pueden usar tanto comillas simples como comillas dobles
